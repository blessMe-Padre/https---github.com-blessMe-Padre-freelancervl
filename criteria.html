Критерии качества верстки и js кода.

Критерии — это список требований, собранных на основе опыта сообщества разработчиков, которым должен соответствовать качественно сверстанный проект.

Код соответствует техническому заданию проекта.

Все обязательные пункты технического задания выполнены. Выполнена HTML-разметка всех страниц проекта и всех элементов на этих страницах.

Отсутствует дублирование кода

В разметке отсутствует дублирование кода для одного и того же элемента, с помощью которого элемент отображается в разных местах страницы на разных версиях: мобильной, десктопной, планшетной. Этот критерий не касается элементов, которые скрываются или показываются в разных версиях. В разметке должно быть использовано минимально возможное количество элементов. Не должно быть лишних обёрток и блоков, которые используются для оформления и могут быть заменены на псевдоэлементы.

Отсутствуют типовые ошибки в разметке по методологии.

Отсутствуют следующие ошибки методологии БЭМ: создание элемента без родительского блока, создание элемента для элемента.

Вёрстка проходит тест на переполнение контентом.

Вёрстка не ломается при добавлении в элементы большего количества текста, при использовании картинок с неподходящими размерами, не ломается при изменении количества потоковых блоков: текст не выпадает из блоков, нижерасположенные блоки не скрываются, смещение блоков в потоке сохраняет логику потока (не приводит к нарушению сетки).

Раскладка блоков на странице сделана на флексах и гридах

Гриды используются для основных структурных блоков страницы (шапка, главный контент, подвал) и для создания колонок в контенте страниц. Для остальной раскладки используются флексы: навигация, карточки, пагинация, хлебные крошки и другие.

Подключены правильные шрифты.

Необходимо, чтобы в проекте использовались именно те шрифты, которые указаны в макете. При этом их размеры, цвета, и жирность должны быть равны соответствующим параметрам в макетах и техническом задании.

Нестандартные шрифты подключены локально.

Все нестандартные шрифты должны быть подключены в CSS из папки fonts с помощью правила @font-face. Подключаемые шрифты должны быть в форматах woff2 и woff.

Выполнена ретинизация растровой графики.

Графика должна быть подготовлена как минимум для двух вариантов экранов: с обычной и двойной плотностью пикселей.

Использованы изображения в формате WebP.

Для браузеров, поддерживающих формат WebP, контентные изображения подключаются в этом формате. Для браузеров, не поддерживающих WebP, изображения подключаются в формате jpeg или png.

Кроссбраузерность.

Вёрстка идентично отображается в последних версиях браузеров Chrome, Firefox, Safari. При проверке этого критерия необходимо смотреть на размеры и расположение блоков, внешнее сходство с макетом. Проверять работу анимации, если такая имеется. Допускаются небольшие отличия в отображениях шрифтов.

Отсутствуе транслита

Отсутствует транслит в названиях классов, атрибутах, переменных CSS-препроцессора, названиях примесей и так далее. Для названий необходимо использовать английские слова и термины.

Вложенность селекторов не больше двух уровней.

Вложенность усложняет читаемость кода и делает код хрупким. Хорошим подходом считается вложенность не больше двух уровней. Вложенность считается по CSS на выходе.

При выполнении кода не возникает необработанных ошибок.

При открытии диалогов, загрузки данных и работе с сайтом не возникает ошибок, программа не ломается и не зависает.

Используются обязательные блоки кода.

В любых конструкциях, где подразумевается использование блока кода (фигурных скобок) — вроде for, while, if и switch — блок кода используется обязательно, даже если инструкция состоит из одной строчки.

В коде нет заранее недостижимых участков кода.

Например: неиспользуемый код, невыполнимые условия, или операции после выхода из функции.

Константы нигде в коде не переопределяются.

Константы используются только для чтения, и никогда не переопределяются на всем промежутке жизни программы.

Код является кроссбраузерным.

Код является кроссбраузерным и не вызывает ошибок в разных браузерах и разных операционных системах. При проверке этого критерия, необходимо удостовериться в правильной работе и отсутствии сообщений об ошибках в выполняемых скриптах в последних версиях браузеров: Chrome, Firefox, Safari, Microsoft Edge (на движке Blink).

Обработчики событий добавляются и удаляются своевременно.

Обработчики событий для виджетов добавляются только в момент появления виджета на странице и удаляются в момент их исчезновения. Количество обработчиков, подвешенных на глобальную область видимости, не должно возрастать.

Для вставки пользовательских строк использован textContent.

Для защита от XSS-атак, а также изменения исходных данных, запутывание пользователя и прочее.

Отсутствуют лишние приведения и проверки типов.

Если заранее известно, что в переменной число, то нет смысла превращать переменную в число parseInt(myNumber). Тоже касается и избыточной проверки булевой переменной.

Минимальное обращение к элементам.

Поиск элементов по селекторам делается минимальное количество раз, после этого ссылки на элементы сохраняются.
